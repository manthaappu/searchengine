{{Infobox software | name = FlockDB | logo = | screenshot = | caption = A distributed, fault-tolerant graph database. | collapsible = | author = Nick Kallen, Robey Pointer, John Kalucki and Ed Ceaser from [[Twitter]] | developer = [[Twitter]] | released = {{Start date|April 2010}} | latest release version = | latest release date = | latest preview version = | latest preview date = | frequently updated = | programming language = Scala, Java, Ruby | operating system = | platform = | size = | language = | status = Active | genre = [[Database]] | license = [[Apache License]] | website = http://github.com/twitter/flockdb }} '''FlockDB''' is an [[open source]] distributed, [[fault-tolerant]] [[graph database]] for managing data at webscale.http://gigaom.com/2010/04/12/twitter-open-sources-the-home-of-its-social-graph/ It was initially used by [[Twitter]] to build its [[database]] of users and manage their relationships to one another. The developers claim that it is much simpler than other graph databases since it scales horizontally and is designed for on-line, low-latency, high throughput environments such as websites.http://github.com/twitter/flockdb#readme Since it is still in the process of being packaged for outside of Twitter use, the code is still very rough and hence there is no stable release available yet. FlockDB was posted on [[GitHub]] shortly after [[Twitter]] released its Gizzard framework, which it uses to [[query]] the FlockDB distributed [[datastore]].
