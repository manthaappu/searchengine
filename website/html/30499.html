{{Cleanup|date=August 2008}} The '''Public Key Name Service''' (PKNS) is an award-winning{{cite web|author = KIRA Awards|url=http://www.kiraawards.com/news/217983|title=2008 Award Winners| date=2008-05-08|| accessdate=2008-08-26}} technology which stores and authenticates trust linkages.{{cite web|author = CRN Online|url=http://crncanada.ca/index.php?page=shop.product_details&flypage=shop.flypage&product_id=1909&category_id=75&manufacturer_id=0&option=com_virtuemart&Itemid=56&vmcchk=1|title=TrustMe Introduces Security Application For Online Communication || accessdate=2008-08-26}} The PKNS is accessible via a [[REST]]-based [[API]] which enables applications and web sites to secure communications and interactions by authenticating trust linkages. The PKNS supports both [[XML]] and [[JSON]] response formats.{{cite web|author = TMC|url=http://opensourcepbx.tmcnet.com/topics/development-tools/articles/28831-trustme-api-enables-safe-secure-online-communication.htm|title=TrustMe API Enables Safe, Secure Online Communication| date=2008-05-22|| accessdate=2008-08-26}} Access to the PKNS is provided free using the TrustMe API.{{cite web|author = CrunchBase|url=http://www.crunchbase.com/company/trustme|title=TrustMe Company Profile|| accessdate=2008-08-26}} ==Open Protocol Support== On July 23, 2008, [[TrustMe Security]] announced, at the [[Facebook]] F8 Conference in [[San Francisco]], that it was converting the PKNS to provide compatibility with [[OAuth]].{{cite web|author = Reuters|url=http://www.reuters.com/article/pressRelease/idUS96701+23-Jul-2008+BW20080723|title=TrustMe Announces Support for OAuth Open Protocol| date=2008-07-23|| accessdate=2008-08-26}} ==Advantages== The Public Key Name Service is notable because it addresses key deficiencies of hosted communication encryption services which have been criticized because the message remains stored on the provider's servers.
