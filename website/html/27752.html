'''Order-independent transparency''' or OIT in [[computer graphics]] denotes any technique that can correctly render overlapping semi-transparent objects without having to sort them before they are being rendered.{{cite web | url=http://www.codermind.com/answers/What-is-order-independent-transparency.html | title=What is order independent transparency? | publisher=Codermind | accessdate=2008-10-12}} Candidates for order independent transparency solutions include: * Storage of samples in an accumulation buffer to be sorted at a later time * [[Depth peeling]]{{cite web | url=http://developer.nvidia.com/object/Interactive_Order_Transparency.html | title=Interactive Order-Independent Transparency | date=2001-05-15 | publisher=[[Nvidia]] | first=Cass | last=Everitt | accessdate=2008-10-12}} * Screen door transparency The Sega [[Dreamcast]] games console included hardware support for automatic OIT{{cite web | url=http://msdn.microsoft.com/en-us/library/ms834190.aspx | title=Optimizing Dreamcast Microsoft Direct3D Performance | date=1999-03-01 | publisher=[[Microsoft]]}}. == References == {{Reflist}} [[Category:Computer graphics]] {{Compu-graphics-stub}}
