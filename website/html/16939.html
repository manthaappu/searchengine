{{Orphan|date=February 2009}} {{Infobox Software | name = HooX | logo = | screenshot = | caption = | developer = [[Groupe Bull]] | latest_release_version = | latest_release_date = | operating_system = [[GCOS]] [[VMS]] | genre = [[Java Message Service]] [[Message Oriented Middleware]] [[Enterprise Messaging System]] | license = | website = [http://www.bull.com/servers/gcos8/products/interop8/hoox_pr.htm bull.com] | }} '''HooX''' ('''H'''ost '''o'''bject '''o'''riented conne'''X'''ion) is a [[software]] product line allowing communication between [[Mainframe computer|mainframe]]s and [[Application software|applications]] running on the [[Java EE]] or [[.NET Framework|.NET]] [[Software framework|framework]]s.[http://www.bull.com/servers/gcos8/products/interop8/hoox_pr.htm HooX] The product was announced in [[2001]][http://www.thefreelibrary.com/Bull+Launches+HooX+Java+Connection+Tools+to+Link+Open+Server-based...-a076691068 Bull Launches HooX Java Connection Tools to Link Open Server-based E-business Applications to Legacy Transactions. - Free Online Library] and its general shipping availability in [[2002]].http://www.bull.com/servers/gcos8/events/techsem2002/giroux.pdf The mainframes must run the [[GCOS]] 7, GCOS 8 or [[MVS]] [[operating systems]]. It is based on the [http://www.bull.com/servers/gcos8/products/interop8/GTEA.htm GCOS TCP/IP Enterprise Access] [[Library (computing)|library]]. '''HooX J2EE Connector''' implements the [[Java EE Connector Architecture]] to allow mainframes to communicate with [[Java (programming language)|Java]] applications running on any [[Java EE]]-compliant [[application server]] running on [[IBM AIX (operating system)|AIX]] or [[Microsoft Windows|Windows]]. '''HooX JMS''' does the same thing in a [[asynchronous]] way through the [[Java Message Service]].http://www.bull.com/servers/gcos7/ce7/S-HooXJMSGCOS7_va1.pdf Internally, communication with the Java EE server is achieved through the server's [[Java Message Service|JMS]] feature and a JMS agent on the GCOS side.
