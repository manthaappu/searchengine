{{Notability|date=January 2011}} {{Infobox software | name = KommandCore | logo = [[File:О компании 1.png|100]] | screenshot = [[File:Project screen.jpg|300px]] | caption = Main workspace screen | developer = KommandCore LLC | operating_system = [[Cross-platform]] | genre = [[Project management]] |language = English, German, French, Russian | license = [[Proprietary software|Proprietary]] | Positioning = Web-tool for professional project management | website = [http://kommandcore.com/ru/ KommandCore Company] | released = {{start date|2008|5|01}} }} '''KommandCore''' is the [[SaaS]] [[Web application|web-application]] for agile project and communication management and [[Collaboration|collaboration]]; paid project management system or [[project management software]]. It targets private individuals, small and medium enterprises ([[Small and medium enterprises|SME]]). This web-application allows including an unlimited number of participants into a [[Project|project]], distributing and [[Task management|managing tasks]][http://mashable.com/2010/02/11/kommandcore/ KommandCore Combines Project Management With GTD], section "Genuis Idea"., managing and modeling the [[Information flow diagram|information flow processes]] within the project. It provides the internal set of crossing tools for management, communication [http://www.killerstartups.com/Web-App-Tools/kommandcore-com-an-enhanced-way-to-manage-projects An Enhanced Way To Manage Projects], article at KillerStartups, section "Why Kommandcore.Com Might Be A Killer". and collaboration. ''Service ideology'' is based on [[Organizational communication|horizontal communication]] and agile management on people. The particular emphasis is made on opportunities and functions which allow handling intensive and permanent information flows. The whole explanation of the web-service functionality and development process is given in the book "Some Essays about Management". [http://kommandcore.com/vision.html The KommandCore web-service ideology], Essay 6 "Why KommandCore is like this", p.97. == Criticism == === Disadvantages === * the programming language of the main application is [[Adobe Flash|Flash]] * no integration and interaction possibilities between different accounts. Each account is done as a separate and single unit; * slight [[Customer relationship management|contact management]] opportunities; * no connection and interaction between tasks and [[Milestone|milestones]] (events) in the [[Gantt chart]] (planning screen); * slight connection and [[Interaction|interaction]] tools between the main planning [[Workspace|workspace]] (macro-level) and current and detailed activity workspace (micro-level) within a project. === Advantages === * light mobile versions of the web-application for [[Android (operating system)|Android]], [[iPad]] and [[iPhone]];
